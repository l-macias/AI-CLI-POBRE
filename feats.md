1. Runtime Constitution
   Cada proyecto tiene reglas propias obligatorias.

2. Model Budget Controller
   Usar modelos baratos/free por defecto y premium solo cuando el runtime lo justifique.

3. Premium Approval Gate
   Antes de gastar en modelo caro, el runtime pide aprobación o explica por qué.

4. Validation Feedback Loop
   Los errores de TS/Lint/Build se convierten en nuevos planes controlados.

5. Runtime Decision Ledger
   Cada decisión importante queda registrada: modelo usado, costo, riesgo, razón y resultado.

6. Context Firewall
   El runtime decide qué contexto puede ver el modelo. No todo entra al prompt.

7. Tool Intent Contract
   El modelo no pide ejecutar herramientas directamente; propone intención estructurada.

8. Local Project Memory
   Memoria compactada + retrieval + historial de decisiones por repo.

9. Risk-Based Execution
   No todas las tareas tienen el mismo nivel de permisos.

10. Cheap-first Agentic Workflow
    Planner barato, reviewer fuerte solo si hace falta, coder premium opcional.
